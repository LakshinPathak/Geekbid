01 — Design System

Token Architecture Audit

Critical Bug

--accent Variable Override

In globals.css, --accent is defined twice. The GeekBid value #00FF88 is immediately overwritten by shadcn's 0 0% 96%. Any Tailwind utility using var(--accent) resolves to near-white, not green. The app only works because pages use raw hex text-[#00FF88] everywhere — bypassing the token system entirely.

/* globals.css — currently broken */
--accent: #00FF88; ← set here
--accent: 0 0% 96%; ← shadcn overwrites it!

/* Fix: rename the GB token */
--gb-accent: #00FF88;
Architecture Issue

Dual Token System

The landing page (page.tsx) uses white/opacity patterns throughout (text-white/40, border-white/[0.08]). All inner app pages use the GB design tokens (text-[#8A8A9A], border-[#1E1E2A]). Two parallel systems makes theming, dark/light toggling, and global color changes require touching every file.

Landing (page.tsx)
text-white/40
border-white/[0.08]
bg-white/[0.03]
App pages (feed, jobs…)
text-[#8A8A9A]
border-[#1E1E2A]
bg-[#12121A]

⚠️
3 contrast failures across the app
The muted text token #55556A on surface black falls at 2.7:1 — below the 4.5:1 WCAG AA threshold for normal-sized body text. The white/25 pattern used extensively on the landing page reaches only 1.4:1. Recommended fix: raise --text-muted to #6E6E85 (+WCAG AA).
